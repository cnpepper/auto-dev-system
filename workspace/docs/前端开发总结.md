## 🏗️ 前端架构

### 分层架构

```
┌─────────────────────────────────────────────────────────┐
│  自动生成代码 (client/) - 绝对不要修改！                    │
│  ├─ sdk.gen.ts      (API 调用方法)                       │
│  ├─ types.gen.ts    (类型定义)                           │
│  └─ schemas.gen.ts  (Schema 定义)                        │
└─────────────────────────────────────────────────────────┘
                          ↓ 封装调用
┌─────────────────────────────────────────────────────────┐
│  自定义 Hooks (hooks/) - 封装业务逻辑                       │
│  ├─ useProjects.ts  (项目管理)                           │
│  ├─ useStages.ts    (流程控制)                           │
│  └─ useConfigs.ts   (配置管理)                           │
└─────────────────────────────────────────────────────────┘
                          ↓ 使用
┌─────────────────────────────────────────────────────────┐
│  业务组件 (components/ + routes/) - UI 交互                │
│  ├─ ProjectCard.tsx (项目卡片)                           │
│  ├─ routes/_layout/projects.tsx (项目列表)                │
│  ├─ routes/_layout/projects.$projectId.tsx (项目详情)    │
│  └─ routes/_layout/configs.tsx (配置管理)                │
└─────────────────────────────────────────────────────────┘
```

## 🚀 如何验证前端功能

### 1. 启动后端服务

```bash
cd backend
uv run uvicorn app.main:app --reload
```

### 2. 启动前端开发服务器

```bash
cd frontend
npm run dev
```

### 3. 访问前端应用

浏览器打开：`http://localhost:5173`

### 4. 测试核心功能

#### 4.1 项目管理

1. 点击侧边栏"项目管理"
2. 查看项目列表

## 🎯 技术亮点

1. **类型安全**: 基于OpenAPI自动生成SDK，确保前后端类型一致
2. **Hooks模式**: 使用自定义Hooks封装业务逻辑，组件职责单一
3. **响应式设计**: 使用Tailwind CSS实现响应式布局
4. **状态管理**: 使用TanStack Query管理服务端状态
5. **错误处理**: 统一的错误处理和提示机制

---

## 📝 注意事项

1. **SDK不要修改**: `client/` 目录下的文件是自动生成的，不要手动修改
2. **先启动后端**: 前端依赖后端API，需要先启动后端服务
3. **认证机制**: 需要登录后才能访问项目管理和配置管理页面
4. **实时数据**: 当前使用轮询机制，后续可以改为WebSocket实现实时推送
