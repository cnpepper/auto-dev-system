## 架构概览

本项目采用 **FastAPI + React + OpenAPI 自动生成** 的全栈架构，前端通过 OpenAPI 规范自动生成 SDK，确保类型安全。

### 核心特点

- ✅ **类型安全**：前后端类型自动同步，减少手动维护
- ✅ **代码生成**：API SDK 自动生成，避免手写 API 调用代码
- ✅ **分层清晰**：自定义 Hook → 业务组件 → 页面组件，职责分明
- ✅ **状态管理**：使用 TanStack Query 管理服务端状态
- ✅ **组件复用**：基于 shadcn/ui 的可复用组件库

## 实现个性化需求的四种方式

### 方式 1：自定义 Hook（推荐）⭐

**场景**：需要在多个地方复用某个 API 调用逻辑，或者需要添加额外的业务逻辑（缓存、错误处理、状态管理）

#### 示例：现有 useAuth Hook

```typescript
// hooks/useAuth.ts ✅ 现有代码（很好的例子）
import { useMutation, useQuery, useQueryClient } from "@tanstack/react-query"
import { useNavigate } from "@tanstack/react-router"
import { LoginService, UsersService } from "@/client"

const useAuth = () => {
  const navigate = useNavigate()
  const queryClient = useQueryClient()

  // 使用自动生成的 API 方法
  const { data: user, isLoading } = useQuery<UserPublic | null, Error>({
    queryKey: ["currentUser"],
    queryFn: UsersService.readUserMe,  // ← 直接调用自动生成的方法
    enabled: isLoggedIn(),
  })

  const loginMutation = useMutation({
    mutationFn: (data: AccessToken) => LoginService.loginAccessToken({
      formData: data,  // ← 调用自动生成的方法
    }),
    onSuccess: (response) => {
      localStorage.setItem("access_token", response.access_token)  // ← 业务逻辑
      navigate({ to: "/" })
    },
  })

  return { loginMutation, logout, user, isLoading }
}
```

import { StocksService } from "@/client"

export const formatMarket = (market: string): string => {
  const marketMap: Record<string, string> = {
    SH: "上海",
    SZ: "深圳",
    BJ: "北京",
  }
  return marketMap[market] || market
}

export const formatPrice = (price: number): string => {
  return `¥${price.toFixed(2)}`
}

export const formatPercent = (value: number): string => {
  return `${value > 0 ? '+' : ''}${value.toFixed(2)}%`
}

export const getMarketBadgeColor = (market: string): string => {
  const colorMap: Record<string, string> = {
    SH: "bg-blue-100 text-blue-800",
    SZ: "bg-green-100 text-green-800",
    BJ: "bg-purple-100 text-purple-800",
  }
  return colorMap[market] || "bg-gray-100 text-gray-800"
}

// hooks/useStockSearch.ts ✅ 自定义 Hook（带缓存）
import { useQuery } from "@tanstack/react-query"
import { StocksService } from "@/client"

export function useStockSearch(searchTerm: string, enabled: boolean = true) {
  return useQuery({
    queryKey: ["stocks", "search", searchTerm],
    queryFn: () => StocksService.readStocks({ search: searchTerm, limit: 20 }),
    enabled: enabled && searchTerm.length > 0,  // ← 只有输入内容且启用时才查询
    staleTime: 5 * 60 * 1000,  // ← 缓存 5 分钟
  })
}

### 案例 2：带加载和错误处理的 CRUD 操作

```typescript
// hooks/useUsers.ts ✅ 用户 CRUD Hook
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query"
import { UsersService } from "@/client"

export function useUsers() {
  return useQuery({
    queryKey: ["users"],
    queryFn: () => UsersService.readUsers({}),
  })
}

export function useCreateUser() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: (data: UserCreate) => UsersService.createUser({ requestBody: data }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["users"] })  // ← 刷新列表
      showSuccessToast("用户创建成功")
    },
    onError: (error) => {
      showErrorToast(`创建失败: ${error.message}`)
    },
  })
}

export function useUpdateUser() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: ({ userId, data }: { userId: number; data: UserUpdate }) =>
      UsersService.updateUser({ userId, requestBody: data }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["users"] })
      showSuccessToast("用户更新成功")
    },
    onError: (error) => {
      showErrorToast(`更新失败: ${error.message}`)
    },
  })
}

export function useDeleteUser() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: (userId: number) => UsersService.deleteUser({ userId }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["users"] })
      showSuccessToast("用户删除成功")
    },
    onError: (error) => {
      showErrorToast(`删除失败: ${error.message}`)
    },
  })
}
```

## 注意事项

### ⚠️ 禁止操作

1. **绝对不要修改 `client/` 目录下的自动生成代码！**
   - 每次后端更新都会重新生成，你的修改会被覆盖
   - 所有修改都应该在自定义 Hook 或组件中完成

2. **不要直接在页面组件里调用 XxxService 的方法！**
   - 先封装成自定义 Hook，再在页面中使用
   - 保持组件职责单一，便于复用和测试

3. **工具函数要纯函数化，不要依赖外部状态！**
   - 方便测试和复用
   - 避免副作用

### ✅ 推荐做法

1. **自定义 Hook 的命名规范**
   - 只读查询：`use[Resource]`（如 `useStocks`、`useUsers`）
   - 写操作：`use[Action]`（如 `useSyncStocks`、`useCreateUser`）

2. **使用 TypeScript 的类型**
   - 自动生成的类型定义在 `@/client` 中
   - 不要自己重新定义类型

3. **合理使用 TanStack Query 的缓存**
   - 根据数据变化频率设置合适的 `staleTime`
   - 使用 `invalidateQueries` 手动刷新数据

4. **错误处理要统一**
   - 使用自定义 Hook 封装错误处理逻辑
   - 提供友好的错误提示

5. **组件拆分要合理**
   - 避免组件过大，保持职责单一
   - 可复用的 UI 逻辑封装成业务组件
