# 前端路由跳转问题说明

## 背景
- 页面：`/projects`（父路由 `/_layout/projects`）与子路由 `/projects/$projectId`
- 现象：点击卡片“查看详情”后 URL 变成 `/projects/1`，但页面未切换到详情内容

## 根本原因
- 父路由组件未渲染 `<Outlet />`，导致子路由 `/projects/$projectId` 无法挂载。

## 解决方案
- 在 `routes/_layout/projects.tsx` 引入 `Outlet`，并在检测到路径为 `/projects/:id` 时仅渲染 `<Outlet />`：
  ```tsx
  import { Outlet, useRouterState } from "@tanstack/react-router"

  const isDetail = location.pathname.startsWith("/projects/") && location.pathname !== "/projects"

  if (isDetail) {
    return <Outlet />
  }
  ```

## 验证步骤
1. 启动前端后访问 `/projects`。
2. 点击任意卡片的“查看详情”，URL 应变为 `/projects/<id>`，页面应显示详情内容。
3. 若仍无跳转，检查浏览器控制台报错，并确认父路由已渲染 `<Outlet />`。

## 预防建议
- 任何嵌套路由的父组件都应包含 `<Outlet />`，保证子路由可渲染。
- 在新增/调整路由时，先检查 `routeTree.gen.ts` 的父子关系，确保组件结构匹配。
- 复用 `useRouterState` 判断当前是否处于子路由，避免父列表与子页面同时渲染。
