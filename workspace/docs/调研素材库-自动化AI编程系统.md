## 三、各环节详细需求

### 3.1 产品设计环节

#### 工作流程

| 标签 | 信息内容 |
|------|----------|
| `[需求]` | 调研结果 → AI生成PRD → 人类确认/反馈 → 迭代修改 → 确认 → AI生成原型 → 确认/反馈 → 迭代修改 → 确认 |
| `[需求]` | 多轮交互，逐步细化 |

#### 质量标准

| 标签 | 信息内容 |
|------|----------|
| `[目标]` | 功能点不遗漏 |
| `[目标]` | 覆盖正向流程、反向流程、异常流程 |
| `[目标]` | 包含角色、场景、流程、异常处理 |

#### PRD验收标准

| 维度 | 检查项 |
|------|--------|
| **功能完整性** | 需求调研中的功能点100%覆盖，无遗漏 |
| **角色与权限** | 每个功能明确操作角色、权限边界 |
| **流程完整性** | 正向流程 + 反向流程（回退/撤销）+ 异常流程（错误/超时）+ 边界情况 |
| **交互细节** | 输入项、输出项、校验规则、提示信息、默认值 |
| **数据定义** | 核心字段、格式要求、数据来源、计算逻辑 |
| **非功能需求** | 性能要求、安全要求、兼容性要求 |

#### 原型验收标准

| 维度 | 检查项 |
|------|--------|
| **界面完整** | 所有功能页面/组件均有原型 |
| **状态覆盖** | 初始态、加载态、空态、异常态、成功态 |
| **交互逻辑** | 点击、跳转、弹窗、提示等交互闭环 |
