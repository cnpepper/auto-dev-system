# 错误记录

## SDK服务名错误（发生次数：2）

**问题**：前端导入SDK服务时提示模块不提供导出

**原因**：SDK生成的服务名是 `StockSelectionService`（单数），代码中误写为 `StockSelectionsService`（复数）

**解决**：检查 `sdk.gen.ts` 确认正确的服务名，修改导入语句

## StrategyService构造函数参数错误（发生次数：1）

**问题**：后端接口报500，`TypeError: StrategyService.__init__() takes 2 positional arguments but 3 were given`

**原因**：`StrategyService` 构造函数只接受 `session` 参数，但调用时传入了 `current_user.id` 和 `current_user.is_superuser`

**解决**：将 `session` 参数改为可选，移除不需要的参数


## 组件导入导出错误（发生次数：1）

## 模型字段引用错误（发生次数：1）

## Schema类型错误（发生次数：1）

## 手动fetch请求错误（发生次数：1）

**原因**：使用手动 `fetch` 请求 API，被前端开发服务器拦截返回 HTML

**解决**：改用自动生成的 SDK 服务

## 自选股删除权限检查失败（发生次数：2）

**问题**：删除自选股时报错 "无权访问该自选股"，但查看时没问题

**原因**：current_user.id 是 UUID 类型，API 路由层使用 str(current_user.id) 转成字符串，导致 get_item 方法中比较 item.owner_id (UUID) 和 current_user_id (字符串) 时类型不匹配

**解决**：移除 API 路由层的 str() 转换，修改 Service 层和 Repository 层参数类型从 str 改为 uuid.UUID

## SDK服务名错误（发生次数：3）

**问题**：前端导入SDK服务时提示模块不提供导出 `AnalysisResultsService` does not provide an export`

**原因**：SDK生成的服务名是 `AnalysisService`（单数，包含所有分析相关的方法），代码中误写为 `AnalysisTasksService` 或 `AnalysisResultsService`（复数）

**解决**：检查 `sdk.gen.ts` 确认正确的服务名，修改导入语句

## Repository查询条件错误（发生次数：1）

**问题**：优质股筛选策略查询到0条财务指标数据，但数据库实际有数据

**原因**：调用 `get_by_stock_code_and_report_date(stock_code, "")` 时，report_date 传的是空字符串，repository 会查询 `report_date == ''` 的数据，但数据库里的 report_date 是 "20241231" 这样的格式

**解决**：修改 repository 的 `get_by_stock_code_and_report_date` 方法，当 report_date 为空字符串时，不添加 report_date 的过滤条件

## Schema字段类型不匹配（发生次数：1）

**问题**：成分股列表接口报错，`Input should be a valid integer [type=int_type, input_value=UUID('...'), input_type=UUID]`

**原因**：`BaseModel` 中的 `id` 字段定义为 `uuid.UUID` 类型，但 `IndexStockConsPublic` 中的 `id` 字段被重新定义为 `int` 类型

**解决**：将 `IndexStockConsPublic` 中的 `id` 字段改为 `uuid.UUID` 类型

## 数据过滤逻辑错误（发生次数：1）

**问题**：优质股筛选策略计算复合增长率时找不到数据，日志显示过滤后数据为0条

**原因**：1) 年份提取错误，使用 `report_date[-8:-4]` 对 "20241231" 会取到 "2412" 而非 "2024"；2) 过滤逻辑只保留2022-2024年数据，不保留2021年数据，导致计算复合增长率时缺少起始年份数据

**解决**：修改年份提取为 `report_date[:4]`，过滤逻辑增加2021年数据保留


## 策略判断逻辑错误（发生次数：1）

**问题**：股票600900无财务数据，策略返回 `passed: False`，但被判定为通过筛选

**原因**：策略服务判断逻辑错误，`result.get("result")` 返回的是策略函数返回值（字典），非空字典在Python中被判断为 `True`，即使字典中 `passed: False` 也会被误判为通过

**解决**：修改判断逻辑，检查 `result.get("result", {}).get("passed", False)`

## 年报季志搜索参数名称不匹配（发生次数：1）

**问题**：年报季志管理页面的搜索功能无效

**原因**：前端代码定义参数时使用下划线命名（`stock_code`、`report_date`），但自动生成的 SDK 使用驼峰命名（`stockCode`、`reportDate`）

**解决**：将前端代码中所有参数名从下划线命名改为驼峰命名，包括类型定义、state、searchParams、handleSearch 等所有相关代码

## 数据管理页面缺少下载功能（发生次数：1）

**问题**：资产负债表、现金流量表、利润表页面没有下载功能

**原因**：后端 Service 缺少 `parse_date_range`、`check_data_exists`、`fetch_xxx_with_check` 方法，API 路由缺少 `/fetch` 接口，前端页面缺少下载对话框组件

**解决**：

3. 在前端页面添加下载按钮、下载对话框（包含日期范围输入、跳过已存在数据选项、日期范围格式说明）、handleFetchData 函数
4. 同时修复参数命名问题（下划线改为驼峰）
5. 执行 `npm run generate-client` 重新生成 SDK

## Excel文件解析引擎错误（发生次数：1）

**问题**：上传 `.xls` 文件时报错 "File is not a zip file"

**原因**：`parse_excel_file` 方法强制使用 `openpyxl` 引擎，但 `openpyxl` 只支持 `.xlsx` 格式，`.xls` 格式需要 `xlrd` 引擎

**解决**：根据文件扩展名选择合适的引擎：
```python
engine = "openpyxl" if file_path.suffix == ".xlsx" else "xlrd"
df = pd.read_excel(file_path, engine=engine)
```

## Schema导入位置错误（发生次数：4）

**问题**：生成前端SDK时报错 `ImportError: cannot import name 'XxxCreate' from 'app.models'`

**原因**：Schema（Create/Update/Public）定义在了 models 文件中或从 models/__init__.py 导出了这些 Schema，导致 API 层无法正确从 `app.schemas` 导入

**解决**：遵循正确的架构分层规范：
1. Schema 定义：在 `app/schemas/xxx/xxx_schemas.py` 中定义所有 Schema（XxxCreate/XxxUpdate/XxxPublic/XxxsPublic）
2. Schema 导出：在 `app/schemas/xxx/__init__.py` 中导出所有 Schema
3. 模型定义：在 `app/models/xxx/xxx_model.py` 中只定义数据库模型（继承 BaseModel 和 table=True）
4. 模型导出：在 `app/models/xxx/__init__.py` 中只导出数据库模型和 Base 模型
5. 全局导出：在 `app/models/__init__.py` 中只导出数据库模型，不要导出 Schema

## Excel数字列前导零丢失（发生次数：1）

**问题**：导入 Excel 文件时，代码列的前导零丢失，如 `000001` 变成 `1`

**原因**：pandas 读取 Excel 时，将包含数字的列自动推断为整数类型，导致前导零丢失

**解决**：在 `pd.read_excel` 时使用 `dtype` 参数，强制将代码列作为字符串读取

**代码**：
```python
# 构建dtype字典，指定需要作为字符串读取的列
dtype_dict = {}
for col_idx in [0, 1, 4]:  # 日期、指数代码、股票代码列
    if col_idx < len(df.columns):
        col_name = df.columns[col_idx]
        dtype_dict[col_name] = str

# 读取时指定dtype参数
df = pd.read_excel(file_path, engine=engine, dtype=dtype_dict)
```

**说明**：必须在读取时就指定 dtype，而不是读取后再转换，因为前导零在读取时就已经丢失了

## select函数未导入错误（发生次数：1）

**问题**：后端报500，`NameError: name 'select' is not defined`

**原因**：在 Service 层使用了 `select()` 函数但未导入该函数

**解决**：从 `sqlmodel` 导入 `select` 函数，不要从 `sqlalchemy` 导入

**代码**：
```python
# ✅ 正确
from sqlmodel import Session, select

# ❌ 错误
from sqlmodel import Session
from sqlalchemy import select  # 会导致类型转换问题
```

**说明**：必须从 `sqlmodel` 导入 `select`，SQLModel 的 select 函数会自动将查询结果转换为模型对象，而 SQLAlchemy 的 select 返回 Row 对象，导致类型不匹配。详细说明见文档 `docs/SQLModel与SQLAlchemy的select导入注意事项.md`

## SDK导入错误：client不存在（发生次数：1）

**问题**：前端报错 `The requested module '/src/client/index.ts' does not provide an export named 'client'`

**原因**：前端代码使用了不存在的 `client` 导入（`import { client } from "@/client"`），应该使用自动生成的 Service 方法

**解决**：
1. 将 `import { client } from "@/client"` 改为 `import { StockSelectionService } from "@/client"`
2. 将 `client.POST("/api/v1/data/stock-selection/filter", { body: {...} })` 改为 `StockSelectionService.filterStockSelection({ requestBody: {...} })`

**代码**：
```typescript
// ✅ 正确
import { StockSelectionService } from "@/client"
const result = await StockSelectionService.filterStockSelection({
  requestBody: { exchange: "SSE", limit: 100 }
})

// ❌ 错误
import { client } from "@/client"
const result = await client.POST("/api/v1/data/stock-selection/filter", {
  body: { exchange: "SSE", limit: 100 }
})
```

**说明**：`@/client` 只导出 Service 类和类型，不导出 `client` 对象。所有 API 调用都应该使用自动生成的 Service 方法，参考其他页面（如 stock-info.tsx、constituents.tsx）的实现方式
