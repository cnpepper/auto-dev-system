# 项目开发环境搭建与部署指南

本文档总结了在本地 (Windows) 搭建全栈开发环境的步骤，并提供了后续在 Linux/Windows 服务器上的部署建议。

---

## 1. 本地开发环境搭建 (Windows)

### 1.1 前置准备
在开始之前，请确保已安装以下软件：
- **Docker Desktop**: 用于运行数据库 (必须安装并启动)。
- **Python 3.10+**: 后端开发语言。
- **Node.js (LTS)**: 前端开发环境。
- **Git**: 版本控制工具。

### 1.2 启动数据库 (PostgreSQL)
本项目依赖 PostgreSQL。为了避免在 Windows 上安装繁琐的数据库软件，我们使用 Docker 启动一个容器化的数据库实例。

**启动命令 (PowerShell):**
```powershell
# 使用国内镜像源启动 PostgreSQL 17
docker run -d --name db -p 5432:5432 -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=changethis -e POSTGRES_DB=app docker.1ms.run/postgres:17
```

### 1.3 后端环境配置与启动 (Backend)
后端使用 Python 的 `uv` 工具进行依赖管理和运行。

**步骤 1: 环境修复 (仅首次需要)**
解决 PowerShell 脚本执行权限及 `uv` 环境变量问题：
```powershell
# 允许执行脚本
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 将 Python 脚本目录添加到环境变量 (根据实际 Python 安装路径调整)
$env:Path += ";C:\Users\liu\AppData\Roaming\Python\Python312\Scripts"
[System.Environment]::SetEnvironmentVariable("Path", $env:Path, "User")
```
*注意：执行完上述命令后，建议重启终端。*

**步骤 2: 安装依赖与初始化**
进入 `backend` 目录执行：
```powershell
cd backend

# 安装所有依赖
uv sync

# 执行数据库迁移 (创建表结构)
uv run alembic upgrade head

# 初始化基础数据 (创建默认管理员账号)
uv run python app/initial_data.py
```

**步骤 3: 启动后端服务**
```powershell
# 启动开发服务器 (支持热重载)
uv run fastapi dev app/main.py
```
- **API 文档地址**: http://localhost:8000/docs
- **交互式 API**: http://localhost:8000/redoc

### 1.4 前端环境配置与启动 (Frontend)
前端基于 React + Vite。

**步骤 1: 安装依赖与启动**
新开一个终端窗口，进入 `frontend` 目录执行：
```powershell
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```
- **访问地址**: http://localhost:5173
- 如果生成的访问地址不是5173，需要去配置文件修改配置，否则会cors校验出错。
- 配置如下：FRONTEND_HOST=http://localhost:5174


### 1.5 默认登录账号
系统初始化后会自动创建超级管理员账号：
- **邮箱**: `admin@example.com`
- **密码**: `changethis`

## 2. 生产环境部署指南

### 2.1 Linux 服务器部署 (推荐)
这是本项目最推荐的生产环境部署方式，利用 Docker Compose 编排容器，自动处理 HTTPS (Traefik) 和服务依赖。

**前置要求**:
- Linux 服务器 (Ubuntu/CentOS/Debian)。
- 安装 Docker Engine 和 Docker Compose 插件。
- 域名解析到服务器 IP。

**部署步骤**:

1.  **克隆代码**:
    将代码上传至服务器或使用 git clone。

2.  **配置环境变量 (.env)**:
    修改项目根目录下的 `.env` 文件，关键配置如下：
    ```dotenv
    # 域名配置 (Traefik 会根据此域名自动申请 HTTPS 证书)
    DOMAIN=example.com

    # 环境模式
    ENVIRONMENT=production

    # 安全配置 (务必修改!)
    SECRET_KEY=<生成一个随机的长字符串>
    POSTGRES_PASSWORD=<设置复杂的数据库密码>
    FIRST_SUPERUSER_PASSWORD=<设置复杂的管理员初始密码>

    # 邮件服务 (用于找回密码等功能)
    SMTP_HOST=smtp.example.com
    SMTP_USER=your_email_user
    SMTP_PASSWORD=your_email_password
    ```

3.  **启动服务**:
    ```bash
    # 拉取/构建并后台运行
    docker compose up -d
    ```

4.  **验证**:
    - 前端 Dashboard: `https://dashboard.example.com`
    - 后端 API: `https://api.example.com`
    - 数据库管理 (Adminer): `https://adminer.example.com`

### 2.2 Windows 服务器部署
虽然 Windows Server 支持运行，但为了稳定性与运维便利性，建议优先使用 WSL2 (Windows Subsystem for Linux) 或 Docker Desktop 方式。

**方案 A: Docker Desktop (推荐)**
1.  在 Windows Server 上安装 Docker Desktop。
2.  配置 `.env` 文件 (同 Linux)。
3.  在 PowerShell 中运行 `docker compose up -d`。
4.  *注意*: Windows 防火墙可能会拦截 Docker 网络，需手动放行 80 和 443 端口。

**方案 B: 传统手动部署 (不推荐)**
如果不使用 Docker，需要手动安装和配置以下组件，维护成本极高：
1.  **数据库**: 手动下载安装 PostgreSQL Windows 版并配置用户/数据库。
2.  **反向代理**: 安装 Nginx for Windows，配置反向代理规则指向后端(8000)和前端(静态文件)。
3.  **进程守护**: 使用工具 (如 NSSM 或 PM2) 将 Python 后端和 Node 前端注册为 Windows 服务，实现开机自启。
4.  **HTTPS**: 手动申请 SSL 证书并在 Nginx 中配置。

**总结**: 强烈建议在所有生产环境中使用 **Docker Compose** 方式部署，以保证环境一致性并简化运维工作。

## 3. 扩展开发指南

### 3.1 如何添加新的分析策略
后端支持通过插件化方式添加新的股票分析策略，主要涉及 `backend/app/strategies` 目录。

**步骤 1: 创建策略实现类**
在 `backend/app/strategies/implementations/` 目录下新建 Python 文件（例如 `my_new_strategy.py`），继承 `BaseStrategy` 并实现 `analyze` 方法：

```python
from typing import Any
from app.strategies.base import BaseStrategy

class MyNewStrategy(BaseStrategy):
    # 唯一标识符，对应前端创建策略时的 "算法标识"
    key = "my_new_strategy"
    name = "我的新策略"
    description = "策略的详细描述"

    def analyze(self, symbol: str) -> dict[str, Any]:
        # 在这里编写分析逻辑
        # ...
        return {
            "passed": True,  # 是否通过策略
            "details": {
                "metric_1": 100,
                "message": "符合条件"
            }
        }
```

**步骤 2: 注册策略**
修改 `backend/app/strategies/__init__.py` 文件，导入并注册你的新策略：

```python
from app.strategies.implementations.my_new_strategy import MyNewStrategy

# ... 其他代码 ...

StrategyRegistry.register(MyNewStrategy())
```

完成上述两步后，重启后端服务，即可在前端“分析策略”管理页面中使用该算法标识 (`key`) 创建新的策略配置。
