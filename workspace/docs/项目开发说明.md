# 股票量化分析系统 - 开发说明文档

## 项目概述

本项目是一个基于全栈架构的股票量化分析系统，主要用于个人投资辅助决策。通过第三方接口获取行情数据，执行各种分析策略，生成分析报告，指导投资决策。

**当前定位**：量化数据分析平台，暂未实现交易功能。专注于数据采集、行情监控和策略分析三大核心能力。

---

## 技术栈

### 后端技术栈
- **Web框架**：FastAPI - 高性能异步 Python Web 框架
- **ORM**：SQLModel - 基于 Pydantic 和 SQLAlchemy 的类型安全 ORM
- **数据库**：PostgreSQL 17 - 企业级关系型数据库
- **数据验证**：Pydantic V2 - 数据验证和设置管理
- **定时任务**：APScheduler - 强大的 Python 定时任务调度库
- **认证授权**：JWT (JSON Web Token) 认证
- **数据源**：AkShare - 开源财经数据接口库

### 前端技术栈
- **UI框架**：React 18 + TypeScript
- **构建工具**：Vite - 下一代前端构建工具
- **路由管理**：TanStack Router - 类型安全的文件路由
- **状态管理**：TanStack Query - 服务端状态管理
- **表单管理**：React Hook Form + Zod - 类型安全的表单处理
- **UI组件库**：shadcn/ui + Radix UI
- **样式方案**：Tailwind CSS
- **图表组件**：Recharts
- **HTTP客户端**：自动生成的 OpenAPI SDK

### 部署技术栈
- **容器化**：Docker + Docker Compose
- **反向代理**：Traefik - 自动 HTTPS 证书管理
- **CI/CD**：GitHub Actions

### 目录组织原则

1. **按层级划分**：`api/`、`services/`、`repository/`、`models/` 分别对应四层架构
2. **职责清晰**：
   - `api/` 只负责接口定义和路由
   - `services/` 只负责业务逻辑
   - `repository/` 负责数据访问和转换
   - `models/` 只定义数据实体
   - `schemas/` 定义数据传输对象

## 设计原则

### 1. 高内聚
- 每个模块内部功能高度相关
- 同一模块的代码变更频率相似

### 2. 低耦合
- 模块之间通过接口通信
- 减少模块间的直接依赖
- 使用依赖注入降低耦合度

### 3. 职责单一
- 每个模块只负责一个特定的业务领域
- 每个类只负责一个特定的功能

### 4. 接口隔离
- 为不同客户端提供不同的接口
- 避免大而全的接口

## 开发指南

### 开发规范

#### 后端开发规范
1. **代码风格**：遵循 PEP 8 规范
2. **类型注解**：使用 Python 类型注解（Type Hints）
3. **错误处理**：所有外部 API 调用必须包含异常处理
4. **日志记录**：使用 `core/logger.py` 中的 logger 记录关键操作
5. **数据验证**：使用 Pydantic 模型进行请求/响应验证

#### 前端开发规范
1. **类型安全**：充分利用 TypeScript 类型系统
2. **组件设计**：遵循 React Hooks 最佳实践
3. **状态管理**：服务端状态使用 TanStack Query，本地状态使用 useState
4. **样式规范**：使用 Tailwind CSS 工具类，优先使用 shadcn/ui 组件

#### 数据库迁移规范
```powershell
# 创建迁移
uv run alembic revision --autogenerate -m "描述变更"

# 执行迁移
uv run alembic upgrade head

# 回滚迁移
uv run alembic downgrade -1
```
